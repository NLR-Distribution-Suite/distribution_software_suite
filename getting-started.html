Install Grid Data Models

Grid Data Models (GDM) provides the standardized schemas that every other tool in the suite depends on. It defines the vocabulary for lines, transformers, loads, and all other distribution assets.

# Install from PyPI
$ pip install grid-data-models

# Or clone for development
$ git clone https://github.com/NLR-Distribution-Suite/grid-data-models.git
$ cd grid-data-models
$ pip install -e ".[dev]"
lightbulb

Tip

Check out the GDM Tutorial repo for interactive Jupyter notebooks that walk you through every model type.

3

Build Your First Model

Create a simple distribution system model using GDM's Pydantic-based schemas. Every model is fully validated and serializable out of the box.

# Create a simple distribution bus
from gdm import DistributionBus
from gdm import DistributionLoad

# Define a bus
bus = DistributionBus(
    name="bus_001",
    nominal_voltage=12.47,
    phases=["A", "B", "C"],
)

# Serialize to JSON
print(bus.model_dump_json(indent=2))
info

GDM models are built on Pydantic v2, so you get automatic validation, serialization, and IDE autocompletion for every field.

Run a Simulation with SHIFT

SHIFT is the high-performance simulation engine. It reads GDM models and runs power flow, time-series, and scenario analyses at scale.

# Install SHIFT
$ pip install shift

# Or from source
$ git clone https://github.com/NLR-Distribution-Suite/shift.git
$ cd shift
$ pip install -e ".[dev]"
# Run a basic power flow
from shift import PowerFlowSolver

solver = PowerFlowSolver.from_gdm(system)
results = solver.solve()

print(results.summary())
speed

Performance

SHIFT is optimized for large-scale feeders. It handles 10,000+ node systems with sub-second solve times.

Convert Models with DiTTo

DiTTo (Distribution Transformation Tool) converts between distribution model formats — OpenDSS, CIM, CYME, Synergi, and more. It's the Rosetta Stone for grid data.

# Install DiTTo
$ pip install ditto.py

# Or from source
$ git clone https://github.com/NLR-Distribution-Suite/ditto.git
$ cd ditto
$ pip install -e ".[dev]"
# Convert OpenDSS model to GDM format
from ditto import Store, convert

# Read from OpenDSS
store = Store()
store.read(input_format="opendss", path="./master.dss")

# Write to CIM
store.write(output_format="cim", path="./output/")

Connect AI Agents

The suite is designed to work with AI agents through the Model Context Protocol (MCP). This enables LLMs like Claude or GPT to interact with your grid data and simulations using natural language.

AI Agent Session — Connected via MCP
YOU

"Load the IEEE 123-bus test feeder and run a 24-hour power flow simulation with 15-minute intervals."

AI

Loading IEEE 123-bus system via GDM...

Configuring time-series solver in SHIFT...

Running 96-step power flow simulation...

Complete. Found 3 buses with voltage violations below 0.95 p.u. during peak hours (17:00–19:00). Report saved.

# Install GridAI — the MCP bridge
$ git clone https://github.com/NLR-Distribution-Suite/gridai.git
$ cd gridai
$ pip install -e ".[dev]"
auto_awesome

What is MCP?

The Model Context Protocol is a standard that lets AI models "see" structured data. Think of it as giving your LLM X-ray vision into your grid models, simulation results, and engineering workflows.
